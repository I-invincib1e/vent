# Trademark Policy

"OpenVent" and the OpenVent logo are trademarks of Rushikesh Pawar. This is separate from the code license.

## Why this exists

The code is Apache 2.0 — full permission to use, modify, self-host, and even build a competing commercial
product on top of it. That's a deliberate choice (see `DECISIONS.md` ADR-028): permissive code licensing
maximizes trust and contribution, which matters more at this stage than trying to block forks with a
non-standard license.

What Apache 2.0 does *not* give you is the right to call your fork, host, or service "OpenVent" or use its
logo in a way that implies it's the official project or is endorsed by it. That protection lives here, in
trademark law, the same way it does for Kubernetes, Docker, and most serious open-source infrastructure
projects — not in the code license.

## What's fine, no permission needed

- Forking, modifying, self-hosting, or building a commercial product on the code, under any name that isn't
  confusingly similar to "OpenVent."
- Saying "built on OpenVent," "powered by OpenVent," "compatible with OpenVent" — factual, unmodified
  references to the project.
- Using the name in blog posts, talks, comparisons, tutorials, or docs about the project.
- Contributing back under the project name (that's the point).

## What needs permission

- Naming a fork, hosted service, or commercial product "OpenVent," "OpenVent Cloud," or anything confusingly
  similar (e.g. "OpenVent Pro," "GetOpenVent").
- Using the OpenVent logo on your own product, site, or marketing in a way that implies official status.
- Any use that could reasonably make a user think they're using the official project or an official
  partner when they're not.

If you want to do any of the above, open an issue or email the address in `SECURITY.md` — this is meant to
be permissive for legitimate cases (e.g. a well-labeled community fork), not a blocker.
